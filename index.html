Cooltipz.css

Cool tooltips made from simple, pure, customisable CSS

  • Lightweight
  • Modern
  • Accessible
  • Customisable
  • Simple

Cooltipz.css makes adding tooltips to existing HTML elements easy and simple, without JavaScript.

Current version: v1.6.0

Play with Cooltipz.css on CodePen.

Features

  • Easy and simple tooltips with quick setup
  • Highly customisable
  • Pure CSS, no JavaScript
  • Accessible with aria-label attributes
  • Automatically disables animation if the user prefers redued motion
  • Options to use classes or data- attributes
  • Options to import via CDN, npm or yarn
  • Options to import SCSS (npm and yarn only) or CSS stylesheets (expanded or minified)
  • Supports a wide range of characters and languages (including emojis, Font Awesome 4 and Font Awesome 5 icons)
  • Uses Autoprefixer when generating the CSS stylesheets and cssnano to minify the minfied stylesheet

Tooltip Size

The default behaviour for a tooltip size is to remain on a single line, no matter the content within it. If you want the content to wrap, use one of the below sizes:

  • Small - data-cooltipz-size="small" / .cooltipz--small
  • Medium - data-cooltipz-size="medium" / .cooltipz--medium
  • Large - data-cooltipz-size="large" / .cooltipz--large
  • Fit - data-cooltipz-size="fit" / .cooltipz--fit
<button data-cooltipz-dir="top" data-cooltipz-size="small" aria-label="Hello there! I am a small tooltip and I will wrap the text within me.">...</button>
<button class="cooltipz--top cooltipz--small" aria-label="Hello there! I am a small tooltip and I will wrap the text within me.">...</button>
<button data-cooltipz-dir="top" data-cooltipz-size="medium" aria-label="Hello there! I am a medium tooltip and I will wrap the text within me. I'm wider thhan the small tooltip.">...</button>
<button class="cooltipz--top cooltipz--medium" aria-label="Hello there! I am a medium tooltip and I will wrap the text within me. I'm wider thhan the small tooltip.">...</button>
<button data-cooltipz-dir="top" data-cooltipz-size="large" aria-label="Hello there! I am a large tooltip and I will wrap the text within me. I'm wider thhan the small and the medium tooltip.">...</button>
<button class="cooltipz--top cooltipz--large" aria-label="Hello there! I am a large tooltip and I will wrap the text within me. I'm wider thhan the small and the medium tooltip.">...</button>
<button data-cooltipz-dir="top" data-cooltipz-size="fit" aria-label="Hello there! I am a 'fit' tooltip and I will wrap the text within me. My width will always be equal to the width of my parent element.">...</button>
<button class="cooltipz--top cooltipz--fit" aria-label="Hello there! I am a 'fit' tooltip and I will wrap the text within me. My width will always be equal to the width of my parent element.">...</button>

No Animation

Animations can be disabled by applying the data-cooltipz-static attribute or the .cooltipz--static class. Cooltipz.css also use the prefers-reduced-motion media query, so when the user's preferences indicate reduced motion, Cooltipz.css automatically disables animation.

<button data-cooltipz-dir="top" data-cooltipz-static aria-label="Hello there!">...</button>
<button class="cooltipz--top cooltipz--static" aria-label="Hello there!">...</button>

Always Visible

Add the data-cooltipz-visible attribute or the .cooltipz--visible class to make the tooltip always show, even if the user isn't hovering or focusing it.

Tooltips can be shown programatically by using the setAttribute() JavaScript function.

<button data-cooltipz-dir="top" data-cooltipz-visible aria-label="I'm always visible!">...</button>
<button class="cooltipz--top cooltipz--visible" aria-label="I'm always visible!">...</button>

Font Awesome

A little setup is required so you can use Font Awesome unicode values on tooltips in your project. You will need to add Font Awesome's font-family name to the --cooltipz-fontawesome CSS custom property in your project's CSS.

If you are using Font Awesome 4, add the below CSS code to your project's CSS:

:root {
  --cooltipz-fontawesome: FontAwesome;
}

If you are using Font Awesome 5, find the relevant Font Awesome 5 font-family name for your project listed here and add the below CSS code to your project's CSS, replacing "Font Awesome 5 Free" with the font-family you need. For example:

:root {
  --cooltipz-fontawesome: "Font Awesome 5 Free";
}
<button data-cooltipz-dir="top" aria-label="Font Awesome: &#xf03e; &#xf200; &#xf188; &#xf007;">...</button>
<button class="cooltipz--top" aria-label="Font Awesome: &#xf03e; &#xf200; &#xf188; &#xf007;">...</button>

Customisation

Cooltipz.css exposes 13 CSS custom properties that can be overriden using your own styles:

CSS Custom Property Description Default Valid values
--cooltipz-bg-color Tooltip background colour #1f1f1f Accepts any valid value for background-color
--cooltipz-text-color Tooltip text colour #fff Accepts any valid value for color
--cooltipz-font-size Tooltip font size 0.75rem Accepts any valid value for font-size
--cooltipz-slide Distance for tooltip to travel into view 6px Accepts any valid value for translate
--cooltipz-border-radius Tooltip border radius 0.125rem Accepts any valid value for border-radius
--cooltipz-timing Time for tooltip to appear 120ms Accepts any valid value for transition-duration
--cooltipz-cursor The cursor shown on the element the tooltip is attached to pointer Accepts any valid value for cursor
--cooltipz-small The width of a size small tooltip 6.25rem Accepts any valid value for width
--cooltipz-medium The width of a size medium tooltip 12.5rem Accepts any valid value for width
--cooltipz-large The width of a size large tooltip 18.75rem Accepts any valid value for width
--cooltipz-delay-show The timed delay for the tooltip to show 0s Accepts any valid value for transition-delay
--cooltipz-delay-hide The timed delay for the tooltip to hide 0s Accepts any valid value for transition-delay
--cooltipz-arrow-size The size of the tooltip arrow 0.3125rem Accepts any valid value for border-width

To override every tooltip in your project, place your CSS custom properties in the :root selector

:root {
  --cooltipz-bg-color: #f00;
  --cooltipz-text-color: #00f;
  --cooltipz-font-size: 1.5rem;
  --cooltipz-slide: 20px;
  --cooltipz-border-radius: 1rem;
  --cooltipz-timing: 1s;
  --cooltipz-cursor: help;
  --cooltipz-small: 4rem;
  --cooltipz-medium: 14rem;
  --cooltipz-large: 25rem;
  --cooltipz-delay-show: 1s;
  --cooltipz-delay-hide: 2s;
  --cooltipz-arrow-size: 0.75rem;
}

Or override specific tooltips

.custom-cooltip {
  --cooltipz-bg-color: #fff;
  --cooltipz-text-color: #000;
  --cooltipz-font-size: 1.5rem;
  --cooltipz-slide: 50px;
  --cooltipz-border-radius: 1rem;
  --cooltipz-timing: 1s;
  --cooltipz-cursor: help;
  --cooltipz-delay-show: 1s;
  --cooltipz-delay-hide: 2s;
  --cooltipz-arrow-size: 0.75rem;
}
.custom-cooltip-two {
  --cooltipz-arrow-size: 1.5rem;
}

Installation

CDN

In the below CDN links:

  • Replace version after the @ with a version listed here (latest version is always recommended). If you always want to get the latest stylesheet, remove @version completely (Not recommended).
  • Replace file with one of the below
    • cooltipz.css (Expanded stylesheet)
    • cooltipz.min.css (Minified stylesheet)
<!-- HTML -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/cooltipz-css@version/file" />
/* Or CSS */
@import url('https://cdn.jsdelivr.net/npm/cooltipz-css@version/file');

Package manager (npm or yarn)

It's best practice to install Cooltipz.css as a dependency rather than a devDependency

First, install it via npm

npm install cooltipz-css --save

Or, install it via yarn

yarn add cooltipz-css

Then, import it into your JavaScript

import 'cooltipz-css';

Or, import the SCSS or CSS into your Sass/SCSS

In the below imports, replace file with one of the options below:

  • src/cooltipz (.scss file)
  • cooltipz.css (Expanded .css)
  • cooltipz.min.css (Minified .css)
/* Webpack */
@import '~cooltipz-css/file';

/* Non webpack */
@import 'path/to/node_modules/cooltipz-css/file';

Beware

Important: Cooltipz.css uses the ::before and ::after psuedo elements to display the tooltips, therefore you must beware of the following:

  • Cooltipz.css will not work on self-closing HTML elements.
  • If the HTML element already has existing ::before or ::after styling, these could conflict with the Cooltipz.css styles and cause unexpected outputs.
